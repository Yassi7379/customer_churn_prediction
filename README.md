# customer\_churn\_prediction

\# 🚨 客户流失诊断与主动预警系统



\## 项目概述

本项目构建了一套从\*\*问题诊断\*\*到\*\*主动干预\*\*的完整数据链路。核心不仅在于通过统计分析定位流失根因，更在于将这些洞察转化为一个\*\*自动化、业务可执行的Power BI预警看板\*\*，帮助团队将“事后分析”变为“事前预防”。



\*\*核心价值\*\*：当高危客户出现时，业务团队能\*\*立即收到提示并采取行动\*\*，而非等待月度报告。



---



\## 🎯 分析目标与闭环设计

本项目遵循 \*\*“描述诊断 -> 规则制定 -> 工具落地”\*\* 的闭环：

1\.  \*\*诊断阶段 (Python)\*\*：回答“客户为什么流失？”—— 通过统计对比与可视化定位关键因素。

2\.  \*\*规则阶段 (逻辑)\*\*：回答“谁即将流失？”—— 基于发现制定可量化的预警规则。

3\.  \*\*落地阶段 (Power BI)\*\*：回答“如何发现并处理？”—— 开发自动化预警看板，推动行动。



---

\## 🔍 第一部分：流失根因诊断 (Python/Jupyter)



这部分工作是整个系统的逻辑基石，为预警规则提供数据依据。



\### 核心成果

\- \*\*完整诊断报告\*\*：\[`customer\_churn\_prediction.html`](./customer\_churn\_prediction.html) （\*\*推荐首先阅读\*\*）

\- \*\*源代码\*\*：\[`notebooks/customer\_churn\_prediction.ipynb`](./notebooks/customer\_churn\_prediction.ipynb)



\### 关键发现（预警规则的来源）

通过群体对比与趋势分析，我们锁定了三个最关键的预警维度：

1\.  \*\*双重高风险\*\*：{国际套餐+客服呼叫次数大于3次的用户，总体流失率占53%，预警客户人数40人}。

2\.  \*\*日间高消费风险\*\*：{日间费用超过40元的客户群体，没有国际套餐的用户，其流失率占39%，预警客户人数308人}。

3\.  \*\*国际套餐风险\*\*：{国际套餐的用户，其流失率占33%，客服呼叫次数<3次，预警客户人数162人}。

4\. \*\*其余高投诉风险\*\*：{预警人数84人}

\### 预警看板三大核心模块

| \*\*1. 全局预警总览\*\* | 展示当前处于“双重高风险客户”、“日间高消费”、“国际套餐用户”、“高投诉风险”状态的预警客户总数。 | 掌握整体风险名单，分配跟进资源。 |
